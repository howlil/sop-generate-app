# Alur SOP dan Evaluasi

## 1. Tim Penyusun

- **Buat/inisiasi proyek sendiri** → Proyek baru berstatus **Draft**.
- **Simpan sebagai draft** (di detail SOP) → Status berubah **Sedang Disusun**.
- **Selesai** (tombol di detail SOP) → Status **Siap Dievaluasi** (tanpa melalui Kepala OPD).
- **Request Evaluasi** (dari Daftar SOP) → Status **Diajukan Evaluasi**.

## 2. Tim Evaluasi

- **Tanpa penugasan**: Tim evaluasi langsung memilih SOP dari **Daftar SOP Evaluasi** (SOP dengan status Siap Dievaluasi, Diajukan Evaluasi, Berlaku).
- **Evaluasi per SOP**: Klik SOP → isi form evaluasi (Sesuai / Revisi Biro + komentar).
- **Hasil evaluasi**:
  - **Sesuai** → Status SOP menjadi **Dievaluasi Tim Evaluasi**.
  - **Revisi Biro** (ada perubahan/komentar) → Status SOP menjadi **Revisi dari Tim Evaluasi**.

## 3. Verifikasi dan Berita Acara

- Hasil evaluasi diverifikasi melalui **Berita Acara**.
- **1 Berita Acara = 1 kali evaluasi OPD** (satu batch per OPD).
- **Urutan tanda tangan Berita Acara**:
  1. **Biro Organisasi** TTD dulu.
  2. Setelah itu **Tim Penyusun** TTD.

## 4. Kepala OPD — hanya menandatangani SOP

- **Tugas Kepala OPD hanya satu:** menandatangani SOP (TTE). Tidak ada pemeriksaan, Setuju/Tolak, atau pengajuan.
- SOP dengan status **Dievaluasi Tim Evaluasi** (hasil evaluasi = Sesuai) → ditandatangani oleh **Kepala OPD** (TTD elektronik) → status berubah **Berlaku**.
- Di aplikasi: di Detail SOP (dari Daftar SOP), tombol **Mengesahkan** (TTE) muncul ketika status SOP = Dievaluasi Tim Evaluasi atau Terverifikasi dari Biro Organisasi.
